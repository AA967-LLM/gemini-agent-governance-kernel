Project Philosophy

The "Civilization" Manifesto: Evolving from Single Agents to Hierarchical Governance

The Problem: Lonely Singletons

Traditional AI agents suffer from:

  • Unchecked hallucinations
  • No adversarial validation
  • Single point of failure
  • Infinite loops with no recovery
  • Black-box decision making

The Solution: Hierarchical Council

Our approach introduces organizational structure:

  • Governance > Intelligence: Smart models need smarter controls
  • Specialized Roles: Architects, Validators, Mediators
  • Adversarial Checks: Security agents have veto power
  • Self-Healing: Automatic deadlock resolution
  • Complete Visibility: Real-time TUI flight recorder

Weighted Authority

Lead Architect (Gemini Pro): 3.0 weight for complex reasoning. Validator (Groq): 1.0 weight with veto power for security.

Adaptive Routing

Intelligent model selection: Flash for trivial tasks, Groq for validation, Pro for architecture. 60% cost reduction achieved.

Self-Healing

Mediator agents spawned automatically on deadlocks. Chain-of-Verification analysis for conflict resolution.

Consensus Calculation
def calculate_weighted_consensus(self, agent_results: List[Dict]) -> Dict:
    """Calculate weighted consensus with veto power"""
    total_weight = 0
    weighted_score = 0
    veto_triggered = False
    
    for result in agent_results:
        weight = result.get('agent_weight', 1.0)
        verdict = result.get('verdict', 'UNKNOWN')
        confidence = result.get('confidence', 0.0)
        
        # Check for veto (FAIL verdict from validator with veto power)
        if verdict == 'FAIL' and result.get('has_veto', False):
            veto_triggered = True
            return {
                'decision': 'FAIL',
                'confidence': confidence,
                'reason': f'Veto by {result.get("agent_name")}',
                'veto_details': result.get('reasoning')
            }
        
        # Convert verdict to numerical score
        if verdict == 'PASS':
            score = confidence
        elif verdict == 'WARN':
            score = 0.5 * confidence
        else:  # FAIL
            score = 0
        
        total_weight += weight
        weighted_score += score * weight
    
    if total_weight == 0:
        return {'decision': 'ERROR', 'confidence': 0.0}
    
    final_confidence = weighted_score / total_weight
    
    # Determine decision based on confidence threshold
    if final_confidence >= 0.7:
        decision = 'PASS'
    elif final_confidence >= 0.4:
        decision = 'WARN'
    else:
        decision = 'FAIL'
    
    return {
        'decision': decision,
        'confidence': final_confidence,
        'weighted_score': weighted_score,
        'total_weight': total_weight
    }
Adaptive Routing Logic
class AdaptiveRouter:
    def route_task(self, task_description: str, 
                   estimated_complexity: int = None) -> RoutingDecision:
        """Intelligent routing based on complexity and cost"""
        
        # Auto-detect complexity if not provided
        if estimated_complexity is None:
            estimated_complexity = self._estimate_complexity(task_description)
        
        # Get current resource status
        token_status = self.token_manager.get_status()
        gemini_budget_used = token_status['gemini']['spent_today'] / token_status['gemini']['daily_limit']
        
        # Routing decision matrix
        if estimated_complexity <= 2:
            # Trivial task - use cheapest option
            return RoutingDecision(
                provider='google',
                model='gemini-3-flash',
                max_tokens=1000,
                estimated_cost=0.00015,
                reasoning='Low complexity task routed to Flash for cost efficiency'
            )
        
        elif estimated_complexity <= 4:
            # Medium complexity - use Groq for validation
            if gemini_budget_used < 0.8:  # Budget not critical
                return RoutingDecision(
                    provider='groq',
                    model='llama-3.3-70b',
                    max_tokens=4000,
                    estimated_cost=0.0,
                    reasoning='Medium complexity validation task routed to Groq'
                )
            else:
                # Budget critical - still use Groq but with tighter limits
                return RoutingDecision(
                    provider='groq',
                    model='mixtral-8x7b',
                    max_tokens=2000,
                    estimated_cost=0.0,
                    reasoning='Budget-critical: Using Mixtral for validation'
                )
        
        else:
            # High complexity - must use Gemini Pro
            return RoutingDecision(
                provider='google',
                model='gemini-3-pro',
                max_tokens=8000,
                estimated_cost=0.0035,
                reasoning='High complexity architectural task requires Gemini Pro'
            )
